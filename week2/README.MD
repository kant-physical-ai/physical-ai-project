[PA] 과제 2 — 색 기반 물체 인식 추가
===
- 제출 방법: GitHub Repository 링크를 제출 폼에 입력
- 핵심 목표: 과제 1의 실시간 루프 위에 HSV 색 필터링과 contour 검출을 얹어, 물체의 위치를 픽셀 좌표(cx, cy)로 뽑아낸다


# 이번 과제에서 만드는 것
- 과제 1에서 만든 실시간 루프(입력 → 처리 → 출력 → FPS 측정)를 그대로 재사용하고, 그 위에 색 기반 물체 인식을 추가합니다.
```text
웹캠 영상 입력            ← 과제 1 재사용
  → HSV 색 공간 변환
  → 특정 색 마스킹 (예: 빨강)
  → Contour 검출 + 면적 필터링
  → 바운딩 박스 + 중심 좌표(cx, cy) 표시
  → 중심 좌표 / 면적 터미널 출력
  → FPS 표시 · 'q' 종료      ← 과제 1 재사용
  
# 최종화면
[Original + 인식 결과 + FPS]     [Mask]
┌─────────────────────────┐    ┌─────────────────┐
│ FPS: 24.1               │    │                 │
│    ┌──────┐             │    │   ■ (흰색)      │
│    │  ●   │ (340, 250)  │    │   나머지 검정   │
│    └──────┘             │    │                 │
└─────────────────────────┘    └─────────────────┘
   초록 박스 + 빨간 중심점         마스크 결과

터미널: 중심: (340, 250) | 면적: 8000px²
```
- 여기서 뽑아낸 중심 좌표 (cx, cy)는 과제 3의 좌표 변환 입력으로 그대로 쓰이므로, 값이 안정적으로 나오게 만드는 것이 이번 과제의 핵심입니다.

# 전체 구현 흐름
이번 과제는 아래 순서로 기능을 추가합니다. ← 신규가 이번 주에 새로 구현하는 부분입니다.
```text
main()
  ↓
웹캠 연결 / while 루프 / 프레임 읽기    ← 과제 1
  ↓
HSV 변환 → 색 마스킹                    ← 신규
  ↓
Contour 검출 + 면적 필터링 + 중심 좌표  ← 신규
  ↓
바운딩 박스 / 중심점 / 좌표 그리기      ← 신규
  ↓
중심 좌표·면적 터미널 출력             ← 신규
  ↓
FPS 표시 → 원본 + Mask 합쳐 출력       ← 과제 1 구조 재사용
  ↓
'q' 종료 / 카메라·창 정리              ← 과제 1
```
- HSV 마스킹은 원본 컬러 프레임을 입력으로 씁니다. 과제 1의 grayscale/edge 결과가 아니라 cap.read()로 받은 원본 frame을 그대로 HSV로 변환하세요.

# 이번 과제에서 구현해야 하는 함수
- 과제 1의 함수 구조(루프·draw_fps·combine_frames)를 유지한 채, 아래를 새로 추가하고 main()에서 연결합니다.

| 함수             | 입력             | 출력              |역할|
|----------------|----------------|-----------------|---|
| create_mask()  | frame          | mask            |HSV 변환 + 지정 색 마스킹|
| find_objects() | mask           | objects         |contour 검출 + 면적 필터링|
|draw_detection()| frame, objects | frame           |바운딩 박스·중심점·좌표 표시|detect()|
|frame|frame| , mask, objects |인식 전체 파이프라인|
|combine_frames()|frame, mask|combined|원본 + Mask 합치기 (과제 1과 동일 방식)|
|draw_fps()|frame, fps|frame|FPS 표시 (과제 1 재사용)|

- find_objects()가 반환하는 objects는 각 물체의 (cx, cy, area, bbox) 튜플 리스트로 설계하면 이후 단계에서 다루기 편합니다.
