===
- 제출 방법: GitHub Repository 링크를 제출 폼에 입력
- 핵심 목표: 과제 2에서 뽑아낸 중심 픽셀 좌표(cx, cy)를 NumPy 행렬 연산으로 직접 실세계 좌표로 변환하고, 결과를 matplotlib으로 시각화한다


# 이번 과제에서 만드는 것
- 과제 2에서 얻은 물체 중심 좌표 (cx, cy)를 이동 → 스케일 → 회전 순서로 변환해 실세계 좌표로 바꾸고, 픽셀 좌표와 실세계 좌표를 그래프로 함께 보여주는 프로그램을 완성합니다.
```text
웹캠 → 물체 인식 (cx, cy)    ← 과제 2 재사용

  → 변환 행렬 직접 구현 (회전 / 스케일 / 이동)
  → 픽셀 → 실세계 좌표 변환 (행렬 곱)
  → 화면에 픽셀·실세계 좌표 표시
  → matplotlib으로 좌표 실시간 시각화
  → 'a'/'d'로 회전각, '+'/'-'로 스케일 조절
  
  
최종 실행 예시:

[OpenCV 창]                        [matplotlib 창]
┌─────────────────────────┐       pixel(파랑) / world(빨강)
│ FPS: 22.0               │       산점도로 좌표 이력 표시
│ angle: 0                │
│   ┌──────┐              │       물체를 움직이면
│   │  ●   │ px(340,250)  │       점이 따라 찍힘
│   └──────┘ world(0.2,0.1)│
└─────────────────────────┘

터미널: pixel: (340, 250) | world: (0.20, 0.10)
```
- 여기서 만든 변환 함수(rotation_matrix_2d, scale_matrix_2d, translate_2d)와 pixel_to_world는 과제 4의 통합에서 그대로 재사용되므로, 함수 단위로 깔끔하게 만드는 것이 이번 과제의 핵심입니다.


# 좌표 변환 개념 (읽고 시작하세요)
- 이미지 픽셀 좌표는 좌상단이 원점(0,0), 오른쪽·아래로 갈수록 값이 커집니다. 반면 실세계 좌표는 보통 화면 중앙을 원점으로 두고 싶습니다. 그래서 아래 순서로 바꿉니다.
1. 이동(translate): 원점을 이미지 중앙으로 옮깁니다 → (cx - W/2, cy - H/2)
2. 스케일(scale): 픽셀 단위를 실세계 단위(예: 미터)로 줄입니다 → × scale
3. 회전(rotate): 카메라와 실세계의 축 방향 차이를 보정합니다 → 회전 행렬 곱 
- 각 단계는 행렬(또는 벡터 덧셈)로 표현되고, 순서대로 적용하면 픽셀 → 실세계 변환이 됩니다.

# 전체 구현 흐름
```text
main()
  ↓
웹캠 연결 / 프레임 읽기 / 인식        ← 과제 1·2 재사용 → (cx, cy)
  ↓
변환 행렬 함수 (직접 구현)
  ├── rotation_matrix_2d(각도)
  ├── scale_matrix_2d(sx, sy)
  └── translate_2d(점, tx, ty)
  ↓
pixel_to_world(cx, cy)               ← 이동 → 스케일 → 회전
  ↓
화면·터미널에 픽셀/실세계 좌표 표시
  ↓
matplotlib으로 좌표 시각화 (몇 프레임마다 갱신)
  ↓
키 입력으로 각도/스케일 조절 → 'q' 종료
```

# 이번 과제에서 구현해야 하는 함수 

| 함수                   |입력|출력|역할|
|----------------------|---|---|---|
| rotation_matrix_2d() |각도(도)|2×2 배열|회전 행렬 (NumPy 직접 구현)|
| scale_matrix_2d()    |sx, sy|2×2 배열|스케일 행렬|
| translate_2d()       |점, tx, ty|2D 점|이동 변환|
| pixel_to_world()     |(cx, cy), 이미지크기|(wx, wy)|이동→스케일→회전 적용|
| detect() 등           |frame|objects|2주차 인식 재사용|
|main()|없음|없음|인식 → 변환 → 시각화 연결|

- 라이브러리 변환 함수(cv2.getRotationMatrix2D 등)를 쓰지 않고 NumPy로 직접 구현하는 것이 이번 과제의 목적입니다. 행렬이 어떻게 좌표를 바꾸는지 직접 만들어 봐야 이해가 됩니다.
